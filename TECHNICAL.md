# UT Class Finder — Technical Reference

Expo + React Native app for finding rooms on the UT Austin campus. Students type a room code (e.g. `GDC 2.216`), the map flies to that room and highlights its polygon.

---

## Stack

| Concern | Library |
|---|---|
| Framework | Expo (dev client) + React Native, TypeScript |
| Navigation | expo-router (file-based, `app/`) |
| Map | `@rnmapbox/maps` v10 (native Mapbox GL) |
| Auth | `expo-auth-session` (OAuth 2.0 / OIDC + PKCE) |
| Secure storage | `expo-secure-store` (Keychain / Keystore) |

## Data architecture

### Source files

#### `buildings_rooms.geojson`
Raw room polygon data from UT Facilities (54,344 features). Each feature is a polygon for one room with properties:
```
room_id         "0152-02-2.216"   bldg_no + floor + roomNumber
bldg_no         "0152"
building_abbr   "GDC"
description     "GATES DELL COMPLEX"
floor           "02"
room_number     null              always null in this dataset
room_type       "classroom"
area            "623.0"           sq ft
```
Floor values are inconsistent across buildings (e.g. `"01"`, `"001"`, `"GROUND"`, `"W01"` all mean first floor).

#### `campus_buildings.geojson`
Official building footprint polygons (231 features) sourced from UT's ArcGIS map:
- Standard buildings: `Campus_Buildings_view/FeatureServer/0`
- Utility/infrastructure buildings: `Campus_Buildings_High_Security_view/FeatureServer/0`

The "high security" label is UT's data classification for utility buildings (power plants, chilling stations, etc.) — not access-restricted. Both endpoints are publicly queryable. The file was fetched with a bounding-box query covering main campus (`-97.749,30.270,-97.720,30.295`).

Properties: `Building` (= bldg_no, e.g. `"0152"`), `Building_Abbr` (`"GDC"`), `Description`, `Address_Full`.

**7 buildings present in room data but absent from ArcGIS** (not published as shapes): `CCC`, `DI1`, `E13`, `E15`, `E27`, `JCB`, `UTS`.

#### ArcGIS source
```
Portal:    https://trecs.maps.arcgis.com
Web map:   471f5223e6a0445cb7965ac86616e800  ("New Campus Map 2025")
Standard:  https://services9.arcgis.com/w9x0fkENXvuWZY26/arcgis/rest/services/Campus_Buildings_view/FeatureServer/0
High sec:  https://services9.arcgis.com/w9x0fkENXvuWZY26/arcgis/rest/services/Campus_Buildings_High_Security_view/FeatureServer/0
```

### Generated files

#### `buildings.json`
One record per building, generated by `scripts/build-buildings.mjs`:
```ts
{
  id: string        // bldg_no, e.g. "0152"
  abbr: string|null // "GDC"
  name: string|null // "GATES DELL COMPLEX"
  center: [lng, lat]
  footprint: [lng, lat][]  // convex hull of all room vertices
  floors: string[]          // e.g. ["01","02","03","04","05","06","07"]
  roomCount: number
}
```
Run: `node scripts/build-buildings.mjs assets/data/buildings_rooms.geojson`

#### `room-index.json`
Flat array of 54,341 entries for fast room search, generated by `scripts/build-room-index.mjs`:
```ts
{
  room_id:       "0152-02-2.216"
  bldg_no:       "0152"
  building_abbr: "GDC"
  floor:         "02"
  roomNumber:    "2.216"
  center:        [lng, lat]   // polygon centroid, used for camera flyTo
}
```
Run: `node scripts/build-room-index.mjs`

## Search architecture (`src/data/search.ts`)

All search functions are pure/synchronous. The room index is lazy-loaded once (`require()`) and cached.

### Functions

**`searchBuildings(query, limit=8): SearchMatch[]`**  
Parses query into `buildingToken + roomToken`. Returns buildings ranked by: exact abbr → abbr prefix → abbr contains → name contains. Used when the user hasn't typed a room number yet.

**`searchRooms(query, limit=8): RoomMatch[]`**  
Returns rooms where `roomNumber.startsWith(roomToken)` within the matched building. Requires both a building token and room token in the query. Uses a `Map<bldg_no, entries>` index built once on first call (O(1) per building lookup instead of scanning 54 k entries).

**`getRoomsInBuilding(bldgNo, token='', limit=8): RoomMatch[]`**  
Like `searchRooms` but takes a building ID directly (used in building state). Empty token returns the first `limit` rooms.

**`resolveRoom(buildingAbbr, roomToken): RoomMatch|null`**  
Exact lookup by abbr + room number. Used for direct resolution (e.g. from a deep link).

**`parseRoomCode(raw): { buildingToken, roomToken }`**  
Splits `"GDC 2.216"` → `{ buildingToken: "GDC", roomToken: "2.216" }`.  
Splits `"gdc2.216"` → same result (compact form also supported).

### Search autocomplete logic (`app/search.tsx`)

```
No building selected:
  query has room token  →  searchRooms(query)        returns RoomMatch[]
  query has no room token →  searchBuildings(query)  returns SearchMatch[]

Building selected (building state):
  any query              →  getRoomsInBuilding(building.id, query)
```

## Map architecture (`src/map/CampusMap.tsx`)

### Data sources

| ShapeSource id | Data file | What it renders |
|---|---|---|
| `campus-buildings` | `campus_buildings.geojson` | Building footprint fills + outlines |
| `all-rooms` | `buildings_rooms.geojson` | Selected room polygon |

Both files are loaded via `Asset.fromModule(...).downloadAsync()` on mount and cached as local URIs. The `all-rooms` source is always mounted (even before a room is selected) so the 54 k-feature GeoJSON is pre-parsed by the time the user selects a room.

### Layer stack (bottom → top within `campus-buildings`)

```
building-fill           All buildings, subtle fill (colors.bgSubtle, 0.7 opacity)
building-outline        All buildings, thin gray line (colors.mist, 1px)
building-selected-fill  Selected building only, orange tint (burntOrange, 0.1 opacity)
building-selected-outline Selected building only, orange border (burntOrange, 2.5px)
```

**Important:** only the first layer (`building-fill`) uses `aboveLayerID="building"` to anchor the group above the base map's building layer (below roads and labels). Sibling layers stack naturally in child order — do NOT add `aboveLayerID` to the others, as sibling references are resolved at registration time and cause ordering bugs.

The `all-rooms` ShapeSource sits above `campus-buildings` in JSX order:
```
selected-room-fill      Filtered to highlightRoomId, orange fill (burntOrange, 0.85)
```

### Camera behavior

Managed via a single `useEffect` watching `[selectedRoom, selectedBuilding]`:

| State | Camera |
|---|---|
| Room selected | `setCamera({ center: room.center, zoom: 19, flyTo })` |
| Building selected, no room | `setCamera({ center: building.center, zoom: 17, flyTo })` |
| Both cleared (back pressed) | `fitBounds(CAMPUS_BOUNDS, padding: 32, duration: 400)` |
| Initial mount (both null) | No imperative call — declarative `bounds` prop handles it |

The `hasHadSelection` ref prevents the fitBounds reset from firing on initial mount.

### Map style

`Mapbox.StyleURL.Light` (`mapbox://styles/mapbox/light-v11`). Chosen over Street because it has no 3D building extrusions, minimal building fills, and cleaner typography that doesn't compete with the custom building layers. To switch to a fully building-free base, replace with a custom Mapbox Studio style URL in `CampusMap.tsx` line ~118.

## UI states (`app/search.tsx`)

### Normal state
- Search icon `⌕` in bar
- Typing a building name → building autocomplete
- Typing `"GDC 2"` → room autocomplete (building + room token detected)

### Building state (building tapped in autocomplete)
- `←` back button + orange building chip (e.g. `GDC`) in search bar
- Dropdown immediately shows first 8 rooms in that building
- Typing filters rooms within that building only
- Map: building highlighted with orange outline + faint fill

### Room state (room tapped in building state)
- Map flies to room polygon, highlights it orange + shows label
- Building highlight stays visible for context
- `←` goes back to building state (not all the way to normal)

### Back button behavior
```
selectedRoom set   →  clear room, reopen room list (stay in building state)
only building set  →  clear building, reset camera to campus
```

## Authentication (`src/auth/AuthContext.tsx`)

UT EID login via OAuth 2.0 / OIDC with PKCE in the system browser.

- `UT_OAUTH_ENABLED=false` → mock session, 8-hour expiry, app fully usable
- `UT_OAUTH_ENABLED=true` → real UT SSO flow via `expo-auth-session`

Session is stored in device keychain (`expo-secure-store`). Redirect URI: `utclassfinder://redirect` — must be registered with UT ITS.

**Open item:** UT ITS has not yet provisioned the OAuth client. Endpoints and client ID are placeholders in `.env`.

## Running the app

```bash
npm install
cp .env.example .env   # fill in tokens
npx expo run:ios       # or: npx expo run:android
```

Requires a dev client (native Mapbox module — Expo Go won't work).
